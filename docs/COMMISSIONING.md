# Пусконаладка и подключение к мастеру (ESP32 slave)

## 1. Назначение
Этот документ описывает минимально необходимые шаги для запуска slave-узла, проверки модулей и интеграции с мастером по Modbus RTU.

## 2. Аппаратные подключения

### 2.1 RS485 (Modbus RTU)
- UART2 RX: GPIO25
- UART2 TX: GPIO26
- UART2 RTS (DE/RE): GPIO27
- Параметры линии: 19200, 8N1
- slave_id: из NVS, диапазон 1..247

### 2.2 I2C
- SDA: GPIO21
- SCL: GPIO22
- На I2C подключены: ADS1115, DS3231

### 2.3 Исполнительные механизмы
- RLL400 OPEN: GPIO18
- RLL400 CLOSE: GPIO19
- 3-way valve OPEN: GPIO32
- 3-way valve CLOSE: GPIO33

### 2.4 Освещение (релейное)
- Relay 1: GPIO4
- Relay 2: GPIO16
- Логика мощности:
  - 00 -> 0%
  - 10 или 01 -> 50%
  - 11 -> 100%

## 3. Запуск прошивки
1. Подайте питание и убедитесь, что RS485 и I2C устройства физически подключены.
2. После старта инициализируются: NVS, Modbus, I2C, RTC DS3231, RH/ADS1115, RLL400, MAX31865(1/2), клапан.
3. Запускаются две задачи:
- Быстрая (200 мс): позиционирование + управление реле света.
- Медленная (5 с): опрос датчиков, обновление телеметрии, расчет расписания света.

## 4. Что обязательно сделать на стороне мастера

### 4.1 Heartbeat
- Регулярно отправлять валидные Modbus-запросы slave.
- При отсутствии валидных запросов >30 с slave переходит в AUTONOMOUS.

### 4.2 Передача солнечной радиации
- Мастер должен записывать текущую радиацию в регистр `134` (`x10` формат).
- Пока датчик у мастера не подключен, допускается писать `0`.

### 4.3 Порог радиации
- Записать верхний порог в регистр `135` (`x10`).
- Если `radiation >= threshold`, slave принудительно снижает свет до 50% (одно реле выключается), но только внутри активного расписания.

### 4.4 Расписание света
- Настроить слоты `110..121`.
- Время в формате HHMM (`0000..2359`, минуты < 60).
- `ENABLE=0` выключает слот.
- `ON==OFF` при `ENABLE=1` трактуется как 24 часа включено.

### 4.5 APPLY-процедура control-блока
- Порядок:
  1. Записать регистры `100..121`.
  2. Посчитать CRC32 для блока `100..121` и записать в `123..124`.
  3. Записать `APPLY_CMD=1` в `122`.
  4. Проверить `APPLY_STATUS` в `125` и `ACTIVE_CTRL_VERSION` в `126..127`.

## 5. Логика света в системе
Свет управляется только по расписанию и радиации:
1. Если расписание неактивно: оба реле OFF.
2. Если расписание активно и `radiation < threshold`: оба реле ON (100%).
3. Если расписание активно и `radiation >= threshold`: одно реле ON, второе OFF (50%).

## 6. Проверка пусконаладки (чеклист)
1. Modbus связь стабильна, нет таймаутов на чтение/запись.
2. Телеметрия в регистрах `0..8` обновляется.
3. При выключенном расписании свет всегда OFF.
4. При активном расписании и низкой радиации свет 100%.
5. При активном расписании и радиации выше порога свет 50%.
6. При пропадании heartbeat >30 с slave переходит в AUTONOMOUS.
7. При восстановлении 3 успешных циклов связи подряд slave возвращается в REMOTE.

## 7. Примечания
- Сейчас радиация ожидается от мастера через регистр `134`.
- Если мастер не пишет `134`, используется последнее значение (после старта 0).
- Некоторые телеметрические каналы могут быть заполнены нулями, если соответствующие датчики/каналы не задействованы в текущей аппаратной конфигурации.
